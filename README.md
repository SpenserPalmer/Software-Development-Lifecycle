# Software-Development-Lifecycle
CS-250 Software Development Lifecycle 

# How do I interpret user needs and implement them into a program? How does creating user stories help with this?

Being able to interpret user stories means you'll have to understand what the user is trying to accomplish instead of only
focusing on the technical requirements that are being asked. I like to break down user needs into smaller and clearer
goals which translate to features that a program has to be able to support, makes it a lot easier to understand. Being 
able to create user stories hepls so much with the requirements because you're able to frame them out visually from
the user's perspective, like "As a user, I want to ___ so that ___." Being able to see something like this makes it so
much easier to pick which functions you want to prioritize during the development phase and makes sure that the final
project is actually useful for the user and not just technically correct.

# How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

The way that I approach developing programs is by making notes of what needs to be done like a TODO list, and breaking 
down all of the requirements into easily digestable pieces and steps to build more incrementally. In this course, I 
was able to use Agile practices which include the sprint planning, daily progress tracking and the retrospectives. In
the future, I'd like to continue using shorter development cycles like sprints, I would like to have regular feedback
loops to better understand what is expected, and iterative improvement. By doing all of this, I will hopefully be able
to catch issues in programs early, help better organization with everything I am working on, and make those larger
projects feel a lot more manageable instead of feeling like some sort of daunting task.

# What does it mean to be a good team member in software development?

Being able to be a good team member in software development means that you should be able to communicate clearly with
others, being reliable for others, and staying open to feedback you may get instead of seeing it as some negative thing. 
You should also be prepared to contribute consistently while being able to support other people on the team when it's
needed. If you're in an agile environment, collaborating with your team should be a very important thing to everyone
on that team, and being adaptable and willing to change based on the input of your team is very important. When you have
strong team members who follow this methodology correctly, it can help the whole team move forward on a lot of things 
very successfully and efficiently!
